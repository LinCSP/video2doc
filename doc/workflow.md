# Рабочий процесс Video2Doc

## Общая схема

Видео-задание описывает **двухэтапный** процесс преобразования видеоматериала в документацию:

```
┌─────────────┐     ┌─────────────────────────────┐     ┌─────────────────┐
│  Видеофайл  │────►│  ЭТАП 1: Транскрибация      │────►│ Скриншоты +     │
│  (.mp4)     │     │  (автоматический)           │     │ Текст transcript│
└─────────────┘     └─────────────────────────────┘     └────────┬────────┘
                                                                 │
                                                                 ▼
┌─────────────┐     ┌─────────────────────────────┐     ┌─────────────────┐
│ Документация│◄────│  ЭТАП 2: Генерация документа│◄────│ Промпт + Kimi   │
│  (doc/*.md) │     │  (AI-ассистент Kimi CLI)    │     │ CLI             │
└─────────────┘     └─────────────────────────────┘     └─────────────────┘
```

## Этап 1 — Транскрибация видео

### 1.1. Входные параметры

| Параметр | Описание | Значение по умолчанию |
|----------|----------|----------------------|
| `VIDEO_PATH` | Путь к исходному видеофайлу | — (обязательный) |
| `INTERVAL` | Интервал извлечения кадров в секундах | 1 |
| `OUT_DIR` | Корневая папка для результатов | `./<имя_видео>/` |

### 1.2. Подэтапы

#### Шаг 1.1 — Извлечение скриншотов (ffmpeg)

```bash
ffmpeg -i "$VIDEO" \
    -vf fps=1/$INTERVAL \
    "$OUT_DIR/screenshots/frame_%04d.jpg" \
    -loglevel error
```

- Кадры сохраняются в формате `frame_0001.jpg`, `frame_0002.jpg` и т.д.
- Качество JPEG — по умолчанию (достаточно для визуального анализа).
- При `INTERVAL=1` получаем 1 кадр в секунду.

#### Шаг 1.2 — Транскрибация речи (faster_whisper)

```python
from faster_whisper import WhisperModel
model = WhisperModel("medium", device="cpu", compute_type="int8")
segments, info = model.transcribe(video, language="ru", beam_size=5)
```

- Модель: `medium` (~1.5 GB при первом запуске).
- Язык: русский (`ru`).
- Выходные файлы:
  - `transcript.txt` — текст с временными метками `[ 0.0s - 5.1s] текст`
  - `transcript.srt` — субтитры в формате SRT

### 1.3. Результат Этапа 1

```
<VIDEO_NAME>/
├── screenshots/
│   ├── frame_0001.jpg
│   ├── frame_0002.jpg
│   └── ... (N кадров, N = duration / INTERVAL)
├── transcript.txt
└── transcript.srt
```

### 1.4. Индикация прогресса в GUI

Приложение должно отображать:
- Общее количество кадров и текущий прогресс ffmpeg.
- Время выполнения и оставшееся время.
- Лог ошибок (если ffmpeg вернул ненулевой код).

## Этап 2 — Генерация документации

### 2.1. Подготовка промпта

Приложение берёт шаблон из редактора и подставляет актуальные пути:

```markdown
Я перевёл видео в текст и картинки {PROJECT_NAME}/*.
Нужно тщательно проанализировать текст и картинки.
И написать документацию в {OUT_DIR}/doc/*
Также необходимые картинки скопировать в {OUT_DIR}/doc/image/*
и встроить в документацию.

Если нужно проанализировать сам проект то {PROJECTS_LIST}
```

### 2.2. Запуск Kimi Code CLI

```bash
kimi \
  --video-path "$VIDEO_PATH" \
  --out-dir "$OUT_DIR" \
  --prompt-file "$OUT_DIR/promt.md"
```

> Примечание: точные аргументы CLI зависят от актуальной версии Kimi Code CLI. Приложение должно позволять пользователю настроить команду запуска.

### 2.3. Результат Этапа 2

```
<VIDEO_NAME>/
└── doc/
    ├── README.md
    ├── requirements.md
    ├── architecture.md
    ├── workflow.md
    ├── gui-spec.md
    ├── wxwidgets-reference.md
    └── image/
        ├── ... (скопированные скриншоты)
```

## Взаимосвязь с `transcribe.sh`

Существующий скрипт `transcribe.sh` уже реализует Этап 1 в виде консольного Bash-скрипта. Задача GUI-приложения — **интегрировать** его логику:

| Компонент | `transcribe.sh` | Video2Doc GUI |
|-----------|----------------|---------------|
| Ввод | Аргументы командной строки | Поля ввода + диалоги |
| ffmpeg | Прямой вызов | Вызов через `wxProcess` |
| whisper | Python inline-скрипт | Python inline-скрипт через `wxProcess` |
| Прогресс | Консольный progress-bar | `wxGauge` + лог-панель |
| Настройки | Жёстко заданы | Редактируемые, сохраняемые |
| Проекты | Не поддерживается | Список `wxListBox` |
| Промпт | Не поддерживается | Редактор `wxTextCtrl` |
| Kimi | Не поддерживается | Интеграция через `wxProcess` |

## Пример полного цикла

```
1. Пользователь выбирает файл "devel.mp4"
2. Интервал = 1 сек
3. Выходная папка = "/home/user/projects/devel"
4. Добавляет доп. проект: "/home/user/projects/bug_report"
5. Редактирует promt.md (если нужно)
6. Нажимает «Запустить полный цикл»

───[ Этап 1: 131 сек ]───────────────────────────
   ffmpeg создаёт 133 скриншота (frame_0001.jpg … frame_0133.jpg)
   faster_whisper генерирует transcript.txt (20 строк)

───[ Этап 2: ~2-5 мин ]───────────────────────────
   Kimi анализирует скриншоты и текст
   Создаёт документацию в devel/doc/
   Копирует ключевые скриншоты в devel/doc/image/

───[ Результат ]──────────────────────────────────
   devel/
   ├── screenshots/     (133 jpg)
   ├── transcript.txt
   ├── transcript.srt
   ├── promt.md
   └── doc/
       ├── README.md
       ├── requirements.md
       ├── architecture.md
       ├── workflow.md
       ├── gui-spec.md
       ├── wxwidgets-reference.md
       └── image/
           ├── wxwidgets-site.jpg
           ├── vscode-transcribe.jpg
           └── ...
```
